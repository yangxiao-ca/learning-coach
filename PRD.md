# AI 学习教练 - 产品规划书

## 一、产品愿景

一个像人类教师一样的 AI 学习辅导工具，能够：
- 接收任意学习材料（文章、博客、书籍）
- 拆解学习规划，一步步引导学习
- 长期追踪进度，建立学习记录
- 费曼技巧检验学习成果

**目标用户**：自己 + 儿子 → 以后扩展到企业/其他孩子

---

## 二、技术方案

### 技术栈
```
纯前端 Web 应用 (HTML + JavaScript + Tailwind CSS)
- 单文件，无需构建
- 本地存储 (IndexedDB 存储学习记录)
- 多模型 API 支持
```

## 三、核心功能

### MVP (最小可行产品)

#### 1. 学习材料输入
- 文本框输入（支持粘贴）
- 文件上传（.txt, .md, .pdf）
- 标题/主题设置
- **大文件智能处理**（见下方）

#### 1.1 大文件处理策略 ⭐

**问题**：PDF/书籍文件大，每次发送慢且费 Token

**解决方案：两阶段处理 + 分段存储**

| 阶段 | 操作 | Token消耗 |
|------|------|-----------|
| **阶段1：快速分析** | 只发送目录/前3000字 → AI生成大纲 | ~1000 tokens |
| **阶段2：按需学习** | 学到某章时，只发送该章内容 | ~500 tokens/次 |

**分段存储结构**：
```javascript
{
  material: {
    type: "pdf",           // pdf/txt/md
    title: "深度学习.pdf",

    // 快速预览 - 用于生成大纲
    preview: "目录\n第1章 绪论...\n第2章 基础...\n(前3000字)",

    // 分段存储 - 学习时按需加载
    chunks: [
      { id: "c1", title: "第1章 绪论", content: "..." },
      { id: "c2", title: "第2章 基础", content: "..." },
      // ...
    ]
  }
}
```

**PDF 文本提取**（浏览器端）：
- 使用 `pdf.js` 纯前端解析
- 提取文本并按章节分割
- 完全本地处理，无需后端

#### 2. 七步学习流程 ⭐

每个知识点/章节都经过完整的七步学习闭环：

```
┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐
│ ①启动 │→  │②框架 │→  │③学习 │→  │④费曼 │→  │⑤考试 │→  │⑥应用 │→  │⑦总结 │
│      │   │      │   │      │   │      │   │      │   │      │   │      │
└──────┘   └──────┘   └──────┘   └──────┘   └──────┘   └──────┘   └──────┘
   ↓          ↓          ↓          ↓          ↓          ↓          ↓
 预评估     知识网络    知识点    自己讲解   练习题    实际行动    笔记
 猜想法    体系连接    原理讲解   检验理解   巩固     产生结果    导出
```

**每步都可跳过**，用户自主控制学习节奏。

##### ① 启动（预评估）
- AI 提问：让你先猜想答案
- 目的：激活已有知识，确定学习起点
- 输出：评估当前水平，调整后续难度
```
🤖 "在学习这章之前，先猜一猜：什么是神经网络？"
✍️ [你的回答]
📊 评估：你有些概念正确，但有些误解...本章将重点关注...
```

##### ② 学习框架（知识网络）
- AI 生成：知识地图、前后关联
- 目的：建立体系，理解知识在整体中的位置
```
📚 本章知识网络：
   前置：线性代数基础
   当前：神经网络基础
   后续：卷积神经网络
   关联：与 perceptron 模型有关
```

##### ③ 学习知识点（原理讲解）
- AI 讲解：当前章节的核心概念
- 特点：根据②的评估调整难度
```
📖 神经网络的核心是...
[详细讲解，配合例子]
```

##### ④ 费曼输出（检验理解）
- AI 提问：用你自己的话解释
- 你讲解后 AI 评估，指出盲区
```
🤖 "用自己的话解释：什么是反向传播？"
✍️ [你的回答]
📊 评估：基本正确，但损失函数部分可以更清晰...
```

##### ⑤ 考试做题（巩固）
- AI 生成练习题
- 你答题后 AI 判分并讲解
```
📝 练习题：
1. [选择题] 下列哪个是激活函数？
   A. Sigmoid  B. Relu  C. 都是

✍️ [你的答案]
📊 判分：正确！解析...
```

##### ⑥ 实际应用（行动）
- AI 建议：如何应用到实际项目
- 可选：你完成应用后分享结果
```
💡 应用建议：
尝试用 Python 写一个简单的感知器，识别手写数字
...
[完成应用] ✅  分享你的成果
```

##### ⑦ 回顾总结（笔记）
- AI 生成：结构化笔记
- 你可修改补充
- 导出 Markdown 格式
```javascript
// 笔记数据结构
{
  stepId: "c1",
  phase: 7,  // 第7阶段
  note: {
    date: "2026-03-26",
    harvest: "收获：理解了感知机的原理",
    mistakes: "错误：混淆了权重和偏置",
    keyPoints: ["要点1", "要点2"],
    connections: ["与线性代数中向量的关系"],
    applications: "可以用来做简单的二分类",
    nextStep: "继续学习多层神经网络"
  }
}
```

## 七、API 调用示例

### 生成学习规划（大文件优化版）

**策略**：只发送预览（目录+开头），不发送全文
```javascript
async function generatePlan(material, goal, model) {
  // 大文件：只发送 preview（约3000字）
  const contentToSend = material.type === 'pdf'
    ? material.preview  // 只发目录+前几章
    : material.content; // 小文件全文发送

  const prompt = `
你是一位专业的教学设计专家。请分析以下学习材料的目录结构，制定详细的学习规划。

材料目录/预览：
${contentToSend}

学习目标：${goal || '请帮我制定'}

请输出 JSON 格式：
{
  "outline": "整体大纲概述",
  "steps": [
    {
      "title": "步骤标题",
      "chunkId": "对应的章节ID（如c1, c2）",
      "content": "简要说明这一步要学什么",
      "keyPoints": ["要点1", "要点2"]
    }
  ],
  "estimatedHours": 预估小时数
}

注意：只需要基于目录结构规划学习步骤，不要深入讲解具体内容。
`;

  return await callModel(model, prompt);
}
```

**Token 节省对比**：

| 场景 | 原方案 | 优化方案 |
|------|--------|----------|
| 300页PDF书 | ~90,000 tokens | ~1,000 tokens |
| 成本（Claude） | ~$1.5 | ~$0.02 |
| 响应速度 | 30-60秒 | 3-5秒 |

### 学习具体步骤（按需加载）

**策略**：学到某章时，才加载该章内容
```javascript
async function learnStep(step, material, model) {
  // 获取该步骤对应的章节内容
  const chunk = material.chunks.find(c => c.id === step.chunkId);

  const prompt = `
你是专业的老师。请讲解以下内容，让${learnerProfile}能够理解。

学习内容：
${chunk.content}

本步骤目标：${step.keyPoints.join('、')}

请用通俗易懂的方式讲解，并举例说明。
`;

  return await callModel(model, prompt);
}
```

**优势**：
- 每次只发送当前章节（~3000-5000 tokens）
- 节省 90%+ 的 Token 成本
- 响应更快

### 七步学习流程 API

```javascript
// ========== ① 启动：预评估 ==========
async function phase1_Start(chapterTitle, learnerProfile, model) {
  const prompt = `
你是专业的老师。学生即将学习"${chapterTitle}"。

学生画像：${learnerProfile}

请先让学生"猜一猜"这个主题的核心内容，激活已有知识。

请输出：
1. 一个引导性问题，让学生猜想
2. 不要直接给出答案，而是激发好奇心

格式：{"question": "引导性问题"}
`;
  return await callModel(model, prompt);
}

async function phase1_Evaluate(userGuess, chapterContent, model) {
  const prompt = `
学生的猜想：${userGuess}

实际内容预览：${chapterContent.substring(0, 1000)}

请评估学生的理解基础：
1. 哪些概念已经理解（正确）
2. 哪些理解有偏差（需要纠正）
3. 学习难度建议：从什么层次开始讲解

格式：JSON，包含基础评估、难度调整、重点关注
`;
  return await callModel(model, prompt);
}

// ========== ② 框架：知识网络 ==========
async function phase2_Framework(chapterContent, fullOutline, model) {
  const prompt = `
请为以下内容建立知识框架，帮助学生理解其在整体知识体系中的位置。

本章内容：${chapterContent.substring(0, 2000)}

全书大纲：${fullOutline}

请输出：
1. 前置知识：需要先掌握什么
2. 当前位置：在整体中的位置
3. 后续连接：会引向什么内容
4. 跨学科关联：与其他领域的关系

用清晰的图表或列表呈现知识网络。
`;
  return await callModel(model, prompt);
}

// ========== ③ 学习：原理讲解 ==========
async function phase3_Learn(chapterContent, evaluation, model) {
  const difficulty = evaluation.difficulty || "中等";
  const focusAreas = evaluation.focusAreas || [];

  const prompt = `
请讲解以下内容，目标是让${difficulty}水平的学习者理解。

内容：${chapterContent}

重点关注：${focusAreas.join("、")}

要求：
1. 通俗易懂，用例子说明
2. 重点突出${focusAreas.join("、")}
3. 逻辑清晰，循序渐进
`;
  return await callModel(model, prompt);
}

// ========== ④ 费曼：检验理解 ==========
async function phase4_Feynman(topic, explanation, model) {
  const prompt = `
学生刚刚学习了"${topic}"，以下是他的解释：

${explanation}

请评估：
1. 理解程度 (完全理解/基本理解/需要加强)
2. 做得好的地方
3. 理解盲区或错误
4. 具体改进建议

用鼓励但明确的语气回复。如果理解有误，请温和指出并解释正确概念。
`;
  return await callModel(model, prompt);
}

// ========== ⑤ 考试：练习题 ==========
async function phase5_Quiz(chapterContent, keyPoints, model) {
  const prompt = `
根据以下内容，生成5道练习题来检验理解。

内容：${chapterContent.substring(0, 2000)}
关键点：${keyPoints.join("、")}

题目要求：
1. 3道选择题（考察概念理解）
2. 1道简答题（考察表达能力）
3. 1道应用题（考察实际运用）

每题包含：题目、选项（选择题）、正确答案、解析。

输出 JSON 格式。
`;
  return await callModel(model, prompt);
}

async function phase5_Grade(userAnswers, questions, model) {
  const prompt = `
学生的答案：${JSON.stringify(userAnswers)}

标准答案：${JSON.stringify(questions)}

请判分并讲解：
1. 每题的正误
2. 错题的详细解析
3. 总体评价和建议
`;
  return await callModel(model, prompt);
}

// ========== ⑥ 应用：实际行动 ==========
async function phase6_Application(chapterContent, learnerProfile, model) {
  const prompt = `

学习内容：${chapterContent.substring(0, 1500)}
学习者背景：${learnerProfile}

建议一个：
1. 具体可行的项目/行动
2. 预计耗时
3. 所需资源
4. 完成标准
5. 可以展示的成果

目标是让学生通过"做"来巩固理解。
`;
  return await callModel(model, prompt);
}

// ========== ⑦ 总结：生成笔记 ==========
async function phase7_Summary(allPhasesResult, model) {
  const prompt = `
基于学生完成的所有学习阶段，生成一份结构化学习笔记。

①启动结果：${allPhasesResult.phase1}
②框架结果：${allPhasesResult.phase2}
③学习内容：${allPhasesResult.phase3.substring(0, 500)}
④费曼检验：${allPhasesResult.phase4}
⑤考试结果：${allPhasesResult.phase5}
⑥应用建议：${allPhasesResult.phase6}

请生成 Markdown 格式的笔记，包含：
1. 标题和日期
2. 知识框架（前置、当前、后续）
3. 核心收获（3-5条）
4. 误区纠正（如有）
5. 关键概念（术语+定义）
6. 知识连接
7. 实际应用
8. 下一步学习

格式清晰，便于复习。
`;
```
